## 一、我为什么选择这个主题

过去做了不少 OTA 测试、天线设计，脑子里对"空口"的理解始终是：**发送端 → 信道 → 接收端，信道是未知的、时变的，物理层设计就是用已知的数学工具（OFDM、MIMO）去对抗未知的信道。**

但 AI-Native Air Interface 这个概念，让我重新思考这个问题：**如果信道未知，能不能让发送端和接收端一起学？** 这不只是算法的改变，而是整个通信系统设计范式的转变。

---

## 二、核心概念：一个根本性的问题

### 传统通信系统的设计逻辑

传统通信系统是这样的：
1. 先建模信道（瑞利衰落、莱斯衰落等）
2. 根据信道模型设计收发机（调制、编码、MIMO）
3. 信道估计 → 均衡 → 解调 → 解码
4. 每个模块独立优化，级联成系统

关键假设：**信道可以被数学模型精确描述**。在 4G、5G 时代，这个假设基本成立，因为信道的主要物理特性（多径、衰落、噪声）已经被充分研究。

**但问题来了：**
- 毫米波和太赫兹信道的散射特性远比 sub-6GHz 复杂
- 6G 要支持的场景（超大规模 MIMO、近场通信、智能反射面）让信道模型的精度严重下降
- 当信道无法被准确建模时，基于模型的设计就开始失效

### AI-Native 的核心思路

Nokia Bell Labs 的 Hoydis 等人在 2021 年发表的奠基性论文"Toward a 6G AI-Native Air Interface"提出的核心问题是：

> **与其用不准确的信道模型来设计收发机，不如让收发机直接从数据中学习最优的通信方式。**

这个思路的本质是：**用数据驱动的学习替代模型驱动的设计**。

## 三、技术架构：从模块化到端到端学习

### 3.1 端到端通信系统（Deep Tx/Rx）

最激进的做法是把整个通信链路建模为一个**端到端可微分的神经网络**：

```
消息 → 神经网络编码器 → 信道（模拟） → 神经网络解码器 → 恢复消息
```

训练目标：最小化端到端误码率（而非分别优化每个模块）

关键工具：
- **信道模拟器**：用神经网络模拟真实信道的特性（可选：物理启发的信道模型）
- **可微分通信链路**：让梯度能从接收端反向传播到发送端
- **信息瓶颈理论**：确保神经网络学到的不是随机映射，而是真正有用的通信策略

### 3.2 Learned Modulation & Coding（学习的调制与编码）

传统做法：
- 调制：QPSK、16QAM、64QAM — 固定的星座图
- 编码：LDPC、Polar — 基于代数理论的固定码

Learned 做法：
- 星座图本身成为可学习的参数
- 编码结构由神经网络搜索得到
- 可以针对**特定信道条件**动态调整调制编码方式（MCS）

这对应了 3GPP Rel-18/19 讨论的"AI-based Link Adaptation"：不再依赖固定的 CQI 表格，而是用 AI 模型直接预测最优 MCS。

### 3.3 Learned Receiver（学习的接收机）

传统接收机是一系列级联的模块：
```
接收信号 → 滤波 → 同步 → 信道估计 → 均衡 → 检测 → 解码
```

每个模块都有简化假设（如线性信道、高斯噪声），这些假设在真实信道中并不完全成立。

Learned Receiver 的思路：用**一个大的神经网络直接完成从原始接收信号到输出比特的映射**，绕过了逐级简化的假设。

典型架构：CNN 或 Transformer 处理时频域信号，替代传统的 DSP 链路。

### 3.4 最核心的问题：信道如何处理？

这里有两条技术路线之争，也是我认为理解 AI-Native Air Interface 最关键的地方：

**路线一：Black Box 信道模拟**
- 直接用神经网络模拟信道，梯度反向传播通过信道
- 问题：需要真实的信道数据来训练信道模型；如果信道模型不准确，通信系统也会出错
- 代表：OFDM 使用虚构信道训练端到端系统

**路线二：Physics-Informed Neural Network（PINN）信道建模**
- 用物理方程（电磁波传播方程）约束神经网络
- 好处：减少了训练数据需求，提高了泛化能力
- 这和 OTA 测试中用物理模型（Friis 方程、3GPP 信道模型）辅助测量的思路一致

**路线三：Modular Learned（模块化学习）**
- 只学习其中某些模块，保留物理层中已经被证明有效的部分
- 比如只学习均衡器，保持 OFDM 和信道估计不变
- 更实用，也是当前产业界的主流做法

## 四、与我专业最相关的部分：近场与太赫兹

### 4.1 近场通信中的 AI-Native

5G NR 和传统 Massive MIMO 假设信道是**平面波、远场**：
- 信号到达角度（AoA）是固定的
- 距离不影响相位响应

但 6G 的超大规模 MIMO（1024+ 天线）和太赫兹频段，信道特性会发生根本变化：
- **近场效应显著**：信号是球面波，距离不同相位不同
- **空间非平稳性**：阵列的不同部分看到不同的信道
- **信道稀疏性变化**：太赫兹频段传播损耗大，多径变少

用传统模型描述这些特性非常复杂，而**神经网络可以从测量数据中直接学习这些非平凡的信道特性**。

这让我联想到 OTA 测试的球面波探头系统：我们在 CATR（紧缩场）里测的是远场特性，但如果要测近场特性的 AI 模型，就需要全新的测试方法。

### 4.2 对 OTA 测试的影响

这是我作为射频工程师最关心的问题。

**现状**：
- OTA 测试衡量的是收发机的"黑盒"性能（TRP、TIS、EVM、ACLR）
- 测试基于标准化的信道模型（SCM、CDL、TDL）

**AI-Native 带来的变化**：
1. 如果调制编码是动态学习的，那么**性能边界变得模糊**——同一个设备在不同信道下的最优传输策略不同
2. OTA 测试需要覆盖更复杂的信道场景，甚至需要**针对 AI 模型的对抗性测试**（类似于 AI 安全中的对抗样本）
3. **信道探测与学习**：AI 系统本身会持续探测信道，OTA 测试可能需要变成"在线"而非"离线"
4. **仿真 vs 实测的 gap**：AI 模型可能过度适应仿真信道，在真实信道中性能下降。OTA 测试需要新的协议来验证这种泛化能力

## 五、最深刻的技术类比：AI-Native = 自适应光学 + 深度学习

我突然意识到这个方向和我熟悉的两个领域有深层联系：

### 类比一：自适应光学（Adaptive Optics）

在望远镜的自适应光学系统中：
- 实时测量波前畸变（Shack-Hartmann 传感器）
- 用变形镜实时补偿大气湍流
- 整个系统形成闭环控制

AI-Native Air Interface 本质上也是：
- **信道探测 = 波前传感**（测量信道状态）
- **神经网络发送/接收 = 变形镜**（实时调整传输策略）
- **端到端学习 = 闭环控制**（整个系统联合优化）

不同的是，通信系统的"大气湍流"（无线信道）变化更快、更不可预测，而且有**信息论极限**作为物理边界。

### 类比二：深度学习在射频领域的另一个应用

做 OTA 测试时，我们经常遇到的问题：
- 非线性功率放大器（记忆效应）
- IQ 不平衡
- 相位噪声

传统做法是用数字预失真（DPD）来补偿。DPD 本质上也是一个**数据驱动的学习系统**——它测量功放的输入输出，学习非线性特性，然后预补偿。

AI-Native Air Interface 是 DPD 思想的**全链路扩展**：不是只补偿一个功放，而是把整个收发链路都纳入学习框架。

## 六、对实际工作的启发

### 6.1 射频工程师的新技能树

AI-Native Air Interface 意味着射频工程师需要：
1. **理解数据驱动方法**：不再只懂电磁仿真和电路仿真，还需要懂神经网络训练和评估
2. **信道知识的重新定位**：信道物理模型仍然重要，但它的作用从"设计依据"变成了"先验约束"
3. **测试方法的扩展**：学习 OTA 测试如何支持 AI 模型验证（不只是 SNR-EVM 曲线，还要测 AI 决策的鲁棒性）

### 6.2 具体的切入点

**我的判断**：在未来 2-3 年内，最先落地的不会是完整的端到端 AI-Native 系统，而是**AI 辅助的链路自适应**（Link Adaptation）。原因：
- 改动最小，不需要颠覆现有空口标准
- 性能收益明显（尤其在复杂信道条件下）
- 可以在现有硬件上通过软件升级实现

这对射频工程师来说意味着：**理解 AI 模型如何做链路决策**，会成为面试和新项目的加分项。

### 6.3 OTA 测试的新挑战

这一点特别实际：
- 如果某个设备用了 AI-Native 收发机，它的性能不再能用固定 MCS 下的 EVM/ACLR 来完整描述
- OTA 测试可能需要增加**AI 性能评估维度**：对抗性信道下的泛化能力、信道探测效率
- 作为测试工程师，理解 AI 模型的黑盒特性会越来越重要

## 七、我的核心洞见

> **AI-Native Air Interface 的本质，是把通信系统的设计逻辑从"模型优先"转变为"数据优先"。**

这和其他领域（计算机视觉、NLP）的深度学习革命是同一底层逻辑：当问题的复杂度超过了人类设计模型的能力上限，就让数据驱动学习来接管。

但关键约束没有消失：**香农极限、物理定律、无线信道的随机性**，这些仍然是硬约束。AI 只是在给定这些约束下，找到更优的映射策略。

从工程哲学的角度看：
- 4G/5G = 人类用数学模型对抗不确定性
- 6G AI-Native = 让机器从数据中学习最优对抗策略
- 最终效果 = 更接近信道容量极限（尤其在信道模型不准确的场景）

**我对这个方向的判断**：值得持续关注，但不要被过度营销忽悠。完整的 AI-Native Air Interface 在 6G 标准中落地还需要时间（可能到 2030+）。更现实的近期机会是**AI 辅助的链路自适应和信道预测**——改动小，收益明确，工程可行。
